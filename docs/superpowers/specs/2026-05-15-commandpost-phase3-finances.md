# CommandPost Phase 3: Finances Module — Design Spec

**Date:** 2026-05-15
**Author:** Phil Smith + Claude
**Status:** Approved

## Overview

Add a full Finances module to CommandPost covering invoice management, expense tracking, revenue visualization, and per-client profitability. Stripe payment links are supported but optional (gated on `STRIPE_SECRET_KEY` env var). PDF invoice export via `@react-pdf/renderer`. Recurring invoices auto-clone via daily cron job.

## Invoice Status Flow

```
Draft → Sent → Paid
              ↘ Overdue (computed: sent + past due_date)
```

- **Draft:** Editable. Can add/remove line items.
- **Sent:** Records `sent_at`. No longer editable. Can mark paid or generate Stripe link.
- **Paid:** Records `paid_at`. Terminal state.
- **Overdue:** Not a stored status. Any invoice with `status = 'sent'` and `due_date < today` is treated as overdue in queries and UI. This avoids needing a background job to flip statuses.

## Invoice Numbers

Auto-generated as `INV-XXXX` where XXXX is zero-padded. On create, query `SELECT invoice_number FROM invoices ORDER BY id DESC LIMIT 1`, parse the number, increment. First invoice is `INV-0001`.

## Invoices CRUD

### List View — `/finances` (default tab: invoices)

- Table: invoice number, client name, status badge, total amount, due date
- Filter by status (all, draft, sent, paid, overdue)
- Summary cards at top: Total Outstanding (sent + overdue amounts), Paid This Month, Total Overdue
- "+ New Invoice" button

### New Invoice — `/finances/invoices/new`

- Select client (dropdown of active clients)
- Line items: dynamic add/remove rows. Each row: description, quantity, unit price, auto-calculated amount. Running total displayed.
- Due date picker
- Recurring toggle: checkbox + recurrence day (1-28) input
- Notes textarea (optional)
- Save as Draft

### Invoice Detail — `/finances/invoices/[id]`

- Header: invoice number, client name (linked), status badge
- Line items table with totals
- Info: due date, sent date, paid date, notes
- Recurring info if applicable
- Actions (contextual based on status):
  - Draft: Edit, Mark Sent, Delete
  - Sent/Overdue: Mark Paid, Generate Stripe Link (if STRIPE_SECRET_KEY set), Download PDF
  - Paid: Download PDF
- Stripe payment link displayed if generated

### Edit Invoice — `/finances/invoices/[id]/edit`

- Only accessible for Draft invoices. Redirects to detail page if not Draft.
- Same form as New, pre-filled.

## Stripe Integration

**Gated on `STRIPE_SECRET_KEY` env var.** When not set, Stripe buttons are hidden. Everything else works normally.

### Generate Payment Link

- Server action calls Stripe API to create a payment link for the invoice amount
- Stores `stripe_payment_link` on the invoice record
- UI displays the link and a "Copy Link" button

### Sync Payment Status

- Server action calls Stripe API to check if payment was completed
- If paid, updates invoice status to 'paid' and records `paid_at` and `stripe_payment_id`
- Triggered manually via a "Check Payment" button on sent invoices with Stripe links
- No webhooks needed for MVP — manual sync is sufficient for a single-user app

### Implementation

- `src/lib/stripe.ts` — Stripe client initialization (checks env var, exports `isStripeConfigured()` and client)
- Uses `stripe` npm package

## Recurring Invoices

### Behavior

- When `is_recurring = 1` and `recurrence_day` is set on an invoice, it serves as a template for auto-cloning.
- Daily cron checks: for each recurring invoice, if the next due date (recurrence_day of current or next month) is within 7 days AND no clone already exists for that month, create a new invoice:
  - Same client, same line items, same notes
  - New invoice number
  - Due date = recurrence_day of the target month
  - Status = Draft (so user can review before sending)

### Cron Script

- `scripts/recurring-invoices.ts` — standalone Node script
- Run via system cron or PM2 cron: `0 0 * * *` (daily at midnight)
- Imports `initDb` and invoice query functions directly
- Logs actions to stdout

## Expenses

### List View — `/finances?tab=expenses`

- Table: date, description, category badge, amount, client name (or "—")
- Filter by category, month picker
- "+ New Expense" button
- Monthly total displayed at top

### New/Edit Expense

- Simple form: expense date, amount, category dropdown, description, optional client dropdown
- Inline on the expenses tab (modal or expandable form, not a separate page)

### Delete

- Confirm dialog, then delete

## Revenue Dashboard — `/finances?tab=revenue`

### Monthly Revenue Chart

- Bar chart showing last 12 months of revenue (paid invoices)
- CSS-based bars (no chart library). Each bar scaled relative to the max month.
- Month labels on x-axis, dollar amounts on bars

### Breakdown Cards

- **YTD Revenue:** Sum of paid invoices this year
- **YTD Expenses:** Sum of expenses this year
- **YTD Profit:** Revenue - Expenses
- **MRR:** Sum of `total_amount` from recurring invoices that are active (have been paid at least once in last 2 months)
- **One-time Revenue:** YTD revenue minus MRR * months elapsed

### Revenue by Client

- Top 5 clients by paid invoice total, with amounts and percentage of total

## Profitability — `/finances?tab=profitability`

### Table

| Client | Revenue | Expenses | Profit | Margin % |
|--------|---------|----------|--------|----------|

- Revenue = sum of paid invoices for client
- Expenses = sum of expenses linked to client
- Profit = Revenue - Expenses
- Margin = Profit / Revenue * 100 (or "—" if no revenue)
- Sorted by profit descending
- Time period selector: This Month, Last 3 Months, YTD, All Time

## PDF Generation

### Route

- `src/app/api/invoices/[id]/pdf/route.ts` — GET handler
- Uses `@react-pdf/renderer` to generate PDF server-side
- Returns PDF as `application/pdf` with `Content-Disposition: attachment`

### PDF Layout

- Header: "INVOICE" + invoice number
- From: "Phil Smith" (or configurable business name from env)
- To: Client name, email
- Line items table: description, qty, unit price, amount
- Total amount
- Due date, status
- Footer: payment instructions or Stripe link if available

### Access

- "Download PDF" button on invoice detail page (Sent, Paid, Overdue statuses)
- Links to `/api/invoices/[id]/pdf`

## Dashboard Integration

### Summary Card

Replace or enhance existing dashboard cards:
- **Outstanding:** Count and total amount of sent + overdue invoices
- **Overdue:** Count of overdue invoices (red if > 0)

### Action Items

All overdue invoices (sent + past due_date) appear as action items:
- Type: `overdue_invoice`
- Title: "Overdue invoice: INV-XXXX for [Client Name] — $X,XXX due [date]"
- Link: `/finances/invoices/[id]`
- Urgency: `red`

### Alert Bar

Overdue invoices appear in the alert bar as red alerts.

## UI Structure

The `/finances` page uses client-side tab navigation (URL search params: `?tab=invoices|expenses|revenue|profitability`). Default tab is invoices.

Tabs are rendered as a row of buttons at the top of the page. Tab content is server-rendered based on the searchParams. This keeps it as a server component with no client-side routing complexity.

## Navigation

The sidebar already has a "Finances" link pointing to `/finances`. No changes needed.

## Dependencies

- `stripe` — Stripe Node SDK (optional runtime dependency, only used when configured)
- `@react-pdf/renderer` — PDF generation
