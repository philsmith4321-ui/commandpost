# Known Residuals — feat/audible-ai-page code review

Source: multi-agent code review run `20260717-093505-0973cb0d` (7 reviewers, artifacts in `/tmp/compound-engineering/ce-code-review/20260717-093505-0973cb0d/` at review time). These findings were accepted-and-recorded rather than fixed in-branch. Everything else actionable from that review was applied and tested on this branch.

- **P1 · maintainability · `src/lib/db.ts` crossed 1000 lines (985 → 1004).** The inline-migration pattern grows this file linearly with every schema change. Recommended: extract the pragma-guarded ALTER blocks into a dedicated migrations module (e.g. `src/lib/db/migrations.ts`) as a standalone refactor — not done mid-feature because a partial extraction (only the two new migrations) would split the migration log across two homes, and a full extraction is a separate reviewable change.
- **P2 · adversarial · `sources_used` can over-report vs the ~14k-char context budget.** Round-robin interleaving (applied) makes truncation degrade categories evenly, but with many categories selected, `sources_used` still counts retrieved chunks, not chunks that survived `generateContent`'s budget cut. Honest accounting needs `generateContent` to report how many chunks its reference builder actually included — a small seam change, deferred.
- **Residual · vector-mode retrieval path is untested against a real Voyage-configured environment.** The cache/interleave/mode logic is unit-tested with mocks and a real in-memory DB, but the first live vector retrieval will happen in production once `VOYAGE_API_KEY` lands. Run one manual `/audible` generation and check the mode badge after enabling it.
- **Residual · no recurring verification that the nginx gates hold.** The runbook's curl checks are a one-time rollout tripwire; a later droplet config edit could silently ungate `/audible`, `/api/audible/`, or `/api/backup`. Cheap option if wanted: append a curl-for-401 assertion to `scripts/deploy.sh`.
- **Residual · fence hardening at the data layer.** `deleteGeneration(db, id)` has no `kind` parameter (both call sites pre-check via kind-guarded gets) and `getKbDocument`/`chunksForDocuments` are unfenced primitives (every current caller is guarded or nginx-gated). Caller discipline is the current enforcement; a future route added without the guard pattern reopens the leak. Consider kind/doc_set-aware signatures if the route surface grows.
- **Residual · metadata oracle.** Ungated `POST /api/generate` returns 400 exactly when a `sourceId` is an Audible doc, letting an unauthenticated caller enumerate which sequential kb ids are Audible (existence only, never content). Accepted for a single-user app.
